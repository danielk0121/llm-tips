# LLM 작업 규칙

## 기본 지침
- **언어**: 모든 응답, 주석, 커밋 메시지는 **한국어**를 사용한다.
- **작업 단위**: 반드시 1개 작업 단위로 진행하고, 작업 1개당 커밋 1개를 사용한다.
- **자동화**: 컴파일이나 린트 에러가 없는지 확인 후 진행한다.

## 작업 규칙
- **커밋 및 푸시**: 모든 작업이 완료되면 변경 사항을 확인하고, 관련 내용을 포함하여 즉시 커밋 및 푸시를 진행한다.
- **코드 스타일**: 기존 코드의 관례를 따르며, 불필요한 리팩토링은 지양한다.
- **보안**: API 키, 토큰 등 민감 정보는 절대 코드에 포함하지 않으며 환경 변수로 관리한다.
- **파일 관리**: `.gitignore` 규칙을 엄격히 준수한다.
